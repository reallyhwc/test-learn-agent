# Personal Finance Agent — 工程结构

## 技术栈
- 后端: Java 17, Spring Boot 3.4, Spring AI 1.1
- 前端: Vue 3, Element Plus, ECharts, Vite
- 构建: Maven Wrapper (无需全局Maven)
- 存储: CSV文件 (零依赖)
- LLM: DeepSeek/OpenAI兼容API
- MCP协议: SSE传输, 2024-11-05版本

## 架构模式
三层架构: Controller → Service → Repository (CsvDataStore)

## 模块清单

| 模块名 | 端口 | 职责 | 关键类 |
|--------|------|------|--------|
| finance-backend | :8080 | REST API+CSV存储 | CsvDataStore/FinanceService/TransactionController/CategoryController/AccountController |
| finance-mcp-server | :8082 | MCP工具层 | FinanceTools(5个@McpTool) |
| finance-agent | :8081 | AI Agent | ChatController/AccountContextBuilder/ChatMemory |
| finance-frontend | :5173 | Vue3 UI | TransactionForm/TransactionList/ChartPanel/ChatPanel/AccountList |

## 数据模型

### Account
- id: Long
- name: String
- type: AccountType (CASH/BANK/CARD)
- balance: BigDecimal
- userId: String

### Transaction
- id: Long
- accountId: Long
- type: TransactionType (INCOME/EXPENSE)
- amount: BigDecimal
- category: String
- subCategory: String
- note: String
- date: LocalDate
- userId: String

### Category
- id: Long
- name: String
- type: TransactionType (INCOME/EXPENSE)
- parentId: Long (null表示一级分类)

## 跨模块变更影响链

Model字段变更 → CSV Schema → 种子数据 → Service校验 → Controller参数 → MCP工具描述 → Agent Prompt → 前端表单/列表/图表

## 两条数据通路

- **CRUD通路**: 前端 → Backend
- **AI通路**: 前端 → Agent → LLM → MCP Server → Backend

## 存储设计

CSV文件在 `data/` 目录，启动时全量加载内存，写入时原子重命名，ReadWriteLock保护并发。
