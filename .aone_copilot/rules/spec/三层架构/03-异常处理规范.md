# 03 异常处理规范

## 异常分类

| 类型 | 异常类 | HTTP 状态码 | 场景 |
|------|--------|------------|------|
| 业务异常 | `IllegalArgumentException` | 400 | 参数校验失败 |
| 系统异常 | `RuntimeException` | 500 | 不可恢复错误 |
| 资源不存在 | 返回 `Optional.empty()` | 404（Controller 判断） | 查询无结果 |

## 各层处理规则

### Controller 层
- 不做 try-catch，让 `GlobalExceptionHandler` 统一处理
- 仅在需要自定义响应时处理特定异常

### Service 层
- 参数校验失败：直接 `throw new IllegalArgumentException("描述")`
- 业务规则违反：直接 `throw new IllegalArgumentException("描述")`

### Repository 层 (CsvDataStore)
- IO 异常：包装为 RuntimeException 抛出，记录 log.error
- 文件损坏：备份损坏文件，返回空数据启动

### MCP 工具层
- 捕获所有异常，返回友好字符串（如 "查询失败，请稍后重试"）
- 禁止抛异常（会中断 MCP 协议）
- userId 校验异常返回错误消息而非抛出

### 前端
- 统一使用 `handleApiError(e, '操作描述')` 处理
- 展示 `ElMessage.error()` 错误提示
