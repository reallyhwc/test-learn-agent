# 01 命名规范

## Java 后端

### 包名
- 格式：`com.example.{模块}.{层级}`
- 示例：`com.example.finance.controller`、`com.example.mcp.tool`、`com.example.agent.controller`

### 类名
- 大驼峰（PascalCase）
- Controller 层：`{资源}Controller`，如 `TransactionController`
- Service 层：`{业务}Service`，如 `FinanceService`
- Repository 层：`{存储}DataStore`，如 `CsvDataStore`
- Model：`{实体名}`，如 `Transaction`、`Category`
- DTO：`{用途}Response` / `{用途}Request`，如 `TransactionResponse`
- 枚举：`{类别}Type`，如 `TransactionType`、`AccountType`

### 方法名
- 小驼峰（camelCase）
- Controller 层用 REST 语义：`listXxx`、`createXxx`、`getXxx`
- Service 层用业务语义：`summarizeTransactions`、`listCategoriesTree`
- Repository 层用数据语义：`findAllXxx`、`findXxxById`、`saveXxx`

### 变量名
- 小驼峰，集合类型加复数或 List/Map 后缀
- 示例：`transactions`、`categoryTree`、`totalByGroup`

### 常量
- 全大写下划线分隔：`TRANSACTION_SCHEMA`、`MAX_MESSAGE_LENGTH`

## 前端 (Vue 3)

- 组件名：大驼峰，如 `TransactionForm.vue`
- ref 变量：小驼峰，如 `filterCategory`、`categoryTree`
- emit 事件：kebab-case，如 `@saved`
- CSS 类名：kebab-case，如 `.tx-form`、`.chart-panel`

## 测试

- 测试类：`{被测类}Test.java`
- 测试方法：`should{预期行为}`，如 `shouldCreateTransaction`、`shouldFilterByCategory`
