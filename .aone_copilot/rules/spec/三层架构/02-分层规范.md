# 02 分层规范

## Controller 层

**职责：** 参数接收、格式校验、XSS 清洗、日志记录、调用 Service、返回结果

**规则：**
- 禁止包含业务逻辑
- 使用 `@RequestParam` 接收查询参数，`@RequestBody` 接收请求体
- XSS 清洗：对用户可控文本字段调用 `XssUtils.sanitize()`
- 日志：记录关键参数，敏感信息用 `LogMaskUtils` 脱敏
- POST 创建端点标注 `@ResponseStatus(HttpStatus.CREATED)`

## Service 层

**职责：** 业务逻辑、参数业务校验、事务编排、调用 Repository

**规则：**
- 禁止直接操作 HTTP 对象（HttpServletRequest 等）
- 参数校验失败抛出 `IllegalArgumentException`
- 日志记录业务关键操作

## Repository 层 (CsvDataStore)

**职责：** 数据读写、内存缓存、CSV 持久化、线程安全

**规则：**
- 禁止包含业务逻辑
- 所有读操作加 `readLock`，写操作加 `writeLock`
- IO 异常包装为 `RuntimeException` 抛出

## 禁止跨层调用

- Controller 不能直接调用 Repository
- Service 不能操作 HTTP 对象
- Repository 不能调用 Service

## 前端分层

- 组件层 (`*.vue`)：UI 渲染、用户交互
- API 层 (`utils/api.js`)：统一 HTTP 请求封装
- 状态层 (`stores/`)：Pinia 全局状态管理
