# 06 前端规范

## 组件结构

```
<template>  <!-- 模板 -->
<script setup>  <!-- 组合式 API -->
<style scoped>  <!-- 局部样式 -->
```

## 状态管理

- 全局状态：Pinia (`useUserStore`)
- 组件状态：`ref()` 响应式变量
- 计算属性：`computed()` 派生数据
- 侦听器：`watch()` 响应变化

## API 调用

- 统一使用 `utils/api.js` 中的 `apiGet`、`apiPost`
- 错误处理统一使用 `handleApiError(e, '操作描述')`
- URL 参数使用 `encodeURIComponent` 编码

## UI 框架

- 组件库：Element Plus
- 图表：vue-echarts + ECharts 按需引入
- 表单验证：Element Plus 内置校验

## 事件与通信

- emit 事件名用 kebab-case
- props 用 camelCase
- 父子组件通信用 props + emit
- 跨组件用 Pinia store

## SSE 流式处理

- ChatPanel 中使用 fetch + ReadableStream
- 支持三种 event：`data`（回答）、`thinking`（推理）、`error`（错误）
- 逐字符渲染，20ms 延迟

## 响应式

- 移动端自动切换 Tab 模式（📊数据 / 💬助手）
- 使用 CSS 媒体查询或 Element Plus 响应式组件
